JQualityVis Help

Creating a Project

A Project represents a software solution and/or system being analysed by JQualityVis and consists of several source code compilation units (source files) and some contextual settings.

  1. Browse to File > Create New Project....
  2. Enter a Project Name and select the Source Root Directory containing the software's source code files.
  3. Select the Source Language to instruct the parser how to interpret the files.
  4. Configure other options such as the additional classpath string and whether to Preload Metric Measurements.
    This may increase the responsiveness of the user interface in exchange for an additional load wait.
  5. Wait for the parsing and analysis processes to complete. A new workspace pane will open upon completion.

Loading an Existing Project File

Projects can be loaded from serialised XML or binary file formats. Please note that metric measurements are not stored in the project file; all metrics must be computed again from the program models stored within.

  1. Browse to File > Open Project....
  2. Use the Browse... finder to select the Project file desired for loading.
  3. Select the relevant import options based on the format of the file. An incorrect selection will yield an error.
  4. Choose whether to Preload Metric Measurements.This may increase the responsiveness of the user interface in exchange for an additional load wait.

Saving a Project File

As with loading Project files, they can be saved by the program in XML or binary file formats, with the option to compress the output file using GZIP compression.

  1. Browse to File > Save Project....
  2. Use the Browse... finder to select the target path in which to save the Project file.
  3. Select the relevant export options based on the format desired of the file. GZIP compression shall reduce the size of the output file if chosen.

Displaying Metric Measurements

Viewing the metric measurements associated with a particular structural program unit and its children is as simple as selecting the unit in the Project Explorer and observing the lower contextual pane.

  • The left portion of the contextual pane displays the metric measurements associated with the selected component and, in some cases, details of its composition.
  • The right portion of the contextual pane displays a tabular listing of the selected component's children and the applicable metrics, along with their measurements when applied to each child component.

Note: The Batch Metric Calculator accessible from the Tools menu can also be used to calculate metrics in batch mode across an entire Project at ease.

Displaying a Visualisation

Visualisations may be activated once a component and an individual metric to measure its constituent component(s) with has been selected from the Quality Analysis pane.

  1. Ensure that a container component (such as a Project, Package or compound Class) is selected in the Project Explorer.
  2. Choose either the tree or fancy metrics selector tab from within the Quality Analysis pane and select the metric to compute.
  3. Select the desired visualisation to show from the second combo box (fancy selector) or the child tree node (tree selector).
  4. The visualisation is now displayed in the workspace pane.

Note: The selected program component can be changed while a metric is selected. Holding SHIFT and clicking multiple program components will add them to the rendered visualisation.

The Breakdown tab within the Quality Analysis pane shows a simple diagram depicting which characteristics of the component are being measured to attain the visualised metric result values.

Managing Project Settings

Projects have several settings associated with them that can be modified by the user. These include the Project's name, creator and a set of Metric Thresholds that are used to determine when to show warnings in the Thresholds tab of the Quality Analysis pane.

  1. Browse to Settings > Project Settings... with a Project workspace pane selected.
  2. Configure the displayed values using the edit boxes and configuration buttons.
  3. Changes will be reflected in the interface upon clicking the Save Entity button and closing the window.

Note: Any metric can be given a threshold range and the default values are attained from that metric's definition in the program's settings files. These defaults can be changed using the Metric Manager.

Managing Metrics and Visualisations

Metrics and Visualisations are the two principle behavioural definitions that govern the functionality that JQualityVis offers. While they are both defined in external configuration files, the program offers a simple management interface to allow a user to add, edit and remove them without having to open a text editor.

  1. Open the desired management window (Edit Metrics... or Edit Visualisations...) from the Settings menu.
  2. Currently available entries are displayed on the left. A form is provided on the right to enter the details required to add or amend an existing entry.
  3. After changes have been made, use the Commit to Disk button to write the list of entries, including their amendments, to the external XML file. Changes will now persist between application starts.

Adding an Entry

  1. Click the New button to clear the form.
  2. Enter the attributes of the new item to be stored.
  3. Select Save Entry to save the item to program memory. Any form validation errors will be displayed.
    Changes will not persist between application starts until changes are committed to disk as above.

Amending an Entry

  1. Select the entry to be amended in the left pane. Its attributes will populate the form on the right.
  2. Amend the attributes displayed in the form as desired.
  3. Select Save Entry to save the item to program memory. Any form validation errors will be displayed.
    Changes will not persist between application starts until changes are committed to disk as above.

Deleting an Entry

  1. Select the entry to be deleted in the left pane.
  2. Select Delete Entry to delete the item from program memory.
    Changes will not persist between application starts until changes are committed to disk as above.

Managing Quality Models

Quality Models must be managed in an external XML-based file as there is currently no way to do so through the program's user interface.

This file is located at config/qualityModels.xml and contains comments within to demonstrate how the individual attributes are computed.
